# Repository Contribution Guidelines

## ⚠️ Important Notice

Please **do not pull or push changes to the repository carelessly**, as doing so may disrupt or break the **entire repository working tree**.

All contributors are expected to follow proper version control practices to maintain the stability and integrity of the project.

## Guidelines

- **Review your changes carefully** before pushing.
- **Pull the latest changes** from the remote repository before starting new work.
- **Avoid pushing directly to protected or shared branches** unless authorized.
- **Resolve merge conflicts properly** before completing a push.
- **Test your changes locally** to ensure they do not break existing functionality.

## Recommended Workflow

1. Pull the latest updates from the master branch.
2. Create a new feature or fix branch.
3. Commit your changes with clear and meaningful messages.
4. Push the branch to the remote repository.
5. Open a Pull Request for review before merging.

Following these practices helps ensure that the repository remains stable and that everyone can collaborate efficiently.

Thank you for contributing responsibly.
